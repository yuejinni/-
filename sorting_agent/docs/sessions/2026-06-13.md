# Session 2026-06-13：分拣机 Python Agent 集成（Phase 0-3 完成）

## 目标与成功标准
用 Python Agent 替代仓库 C# WinForms + EMSAPI + 15 条 SQL Server 存储过程，
新系统与原有系统完全隔离，可并行运行，随时可回滚。

## 范围与约束
- 新增 `sorting_agent/` 模块，与 `customs_server/` 并列，互不依赖
- `customs_server/server.py` 仅 append（try/except 包裹，不影响主服务）
- 不修改任何现有文件（仅 server.py 注册 blueprint）
- SQL Server 使用独立数据库 `SortingAgent`，与原 `EmsSort` 完全隔离

## 已完成（Phase 0-3）

### Phase 0 — 基础设施
- `sorting_agent/schema.sql`：8 张表 + WHILE 初始化 sort_ports/port_lights 各 200 行 + 15 条 sys_config
- `sorting_agent/core/db.py`：pyodbc 连接工厂 + qone/qval/qall/execute
- `sorting_agent/config.example.json`：含 plc_ip / db_conn_str / cloud_url / use_mock_plc

### Phase 1 — 核心业务
- `plc/plc_mock.py`：MockPLC 开发期空实现
- `plc/plc_client.py`：`_plc_lock`（模块级全局锁）/ connect_plc / write_start_plc / write_port_light
- `plc/plc_reader.py`：read_car_status_loop / read_button_loop（打印用 daemon thread 异步）
- `core/scan_handler.py`：handle_scan / handle_manual_scan / handle_pda_scan（含所有 ADR-0002 修正）
- `core/port_manager.py`：try_reassign_overflow(HOLDLOCK) / auto_update_port / get_port_status / port_monitor_loop
- `sync/rule_sync.py`：sync_rules_from_cloud + rule_sync_loop（30s，含 pick_progress MERGE 初始化）
- `sync/event_push.py`：event_push_loop（3s 批量推送，幂等 event_key）

### Phase 2 — Flask API + TCP + 启动
- `api/web_api.py`：12 条路由（Flask :5008）
- `tcp/tcp_server.py`：asyncio TCP :8888，协议 `carno#barcode\r\n`
- `print/print_manager.py`：Jinja2 + pdfkit 面单打印
- `main.py`：6 daemon thread + asyncio TCP 主循环
- `start.bat`：Windows 双击启动

### Phase 3 — 云端接口
- `customs_server/sorting/batch_planner.py`：allocate_ports（1:N 展开、跳51、label_data）
- `customs_server/sorting/agent_api.py`：Blueprint，5条路由，SQLite 存储（sorting_cloud.sqlite3）
- `customs_server/server.py`：注册 sorting_bp（try/except，不影响现有功能）

## 关键决策
- Flask 端口：**5008**（与云端同号，仓库 PC 是独立机器不冲突）
- SQL Server：独立安装在**专用仓库 PC**（SQL Server 2022 Express 免费），与原有机器零依赖
- PLC Mock：`config.json` 设 `use_mock_plc: true` 即可在无 PLC 环境开发测试
- 打印异步：`print_port_label` 在 daemon thread 执行，不阻塞 500ms 按钮轮询
