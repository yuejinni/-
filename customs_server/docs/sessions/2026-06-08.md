# Session 2026-06-08：建档改为"先存本地草稿，再提交 JDY"

## 目标与成功标准
- 单品建档 / 批量建档不再直接调 JDY API
- 点"保存到草稿箱"时生成编码、存 SQLite，用户在草稿箱确认后才提交 JDY
- 草稿箱支持：单条提交、批次整体提交、删除（回滚序号）

## 范围与约束
- **不改** register_product.py、code_gen.py、server.py
- 复用现有 SQLite（sales_cache.sqlite3），新建 product_drafts 表
- 复用 _cache.create_purchase_order() 在批次提交后生成购货订单

## 预期输出

### 后端 register_server.py 改动
| 改动 | 说明 |
|------|------|
| `import uuid` | 批次 ID 生成 |
| `get_next_seq, rollback_seq, generate_ean13` | 新增 code_gen 导入 |

| `GET /register/drafts` | 草稿列表（默认 pending+error） |
| `DELETE /register/drafts/<id>` | 删除草稿（回滚序号） |

- 草稿箱 Tab：按 batch_group 分组，支持"提交本批次"、单条提交/删除

## 方案选择
选择"先存草稿"方案：
- 建档时只做本地操作（快），用户可以批量核查后再提交
- 编码在存草稿时就生成，用户可以提前记录/打印

## 验收方式
1. 单品：填表 → "保存到草稿箱" → 显示编码 → 草稿箱红点出现 → 点提交 → JDY 有商品
2. 批量：AI 解析 → 确认 → Step5 显示 pending 编码 → 草稿箱提交本批次 → JDY 有商品 + 购货订单
3. 删除 pending 草稿 → sequence_counters.json 序号回滚
4. 46001 时草稿箱重试成功
