### 3. 装箱算法重写 + 体积单位修正 + 批次设置 UI（`2f6fb4c`）

**涉及文件：** `sorting/batch_planner.py`、`sorting/agent_api.py`、`templates/jdy.html`

**体积单位：** `jdy_products.length/width/height` 单位为 **cm**（不是 mm），`l×w×h` 直接得 cm³，去掉之前错误的 `/1000`。

**装箱算法重写（`allocate_ports`）：**
- 不再有 `offset` 容差，改为硬限制（不超过大箱上限）
- 不拆分同一 SKU 的数量
- 按订单内商品顺序累积体积，超出大箱上限时封箱开新格口
- 箱型 = `_find_box_type(vol, box_configs)` 自动选最小合适箱

**批次详情显示修复：**
- `cloud_sorting_batches` 新增 `box_configs_json` 列，生成批次时保存
- `/sorting/batch/<batchno>/rules` 返回 `box_configs`
- 前端 `BOX_CAP` 改为从 API 读取，不再硬编码 `500/2000/5000`

**批次设置 UI 优化：**
- 旧配置卡片（占一列的大表单）替换为筛选栏下方紧凑单行条
- 默认值更新为 `40000/85000/198000 cm³`
- 新增「确认」按钮 + `localStorage` 持久化（`saveSortingConfig` / `loadSortingConfig`）
- 切换到批次管理页自动恢复上次保存的配置
- 校验：小箱 < 中箱 < 大箱，不满足时阻止保存
